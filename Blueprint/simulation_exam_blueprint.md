> **Engine Version**: 1.0.0 | **Blueprint Version**: 3.0.0 | **Position ID**: `bma-education-specialist`

> **หมายเหตุการปรับปรุง (v3.0)**: ปรับจาก 40 ข้อ/ชุด × 10 ชุด (v2.0) เป็น **100 ข้อ/ชุด × 5 ชุด** ทุกตารางสัดส่วน (Tier / Difficulty / Blueprint Type / LO / Pattern) ถูกคำนวณใหม่ตามอัตราส่วนเดิม (×2.5) แล้วปัดเป็นจำนวนเต็มให้ผลรวมเท่ากับ 100 ข้อ/ชุดพอดี ธีมของ 5 ชุดยึดตาม Set 1–5 เดิม (กฎหมายหลักการศึกษา / บริหาร กทม. / นวัตกรรมดิจิทัล / หลักสูตร-ประกันคุณภาพ / แผนงานโครงการ) ส่วนธีมเสริมของ Set 6–10 เดิม (ปฐมวัย-ภาคบังคับ, แนวโน้ม, บูรณาการ, ปฏิบัติการ, สมดุลรอบด้าน) ถูกยุบรวมกระจายเข้าไปในหัวข้อย่อยของ 5 ชุดนี้แทน

### Distribution Constraints

```
FOR each simulation_set:
  SUM(all documents) = 100
  
  FOR each document:
    count >= tier.min
    count <= tier.max
    
  SUM(tier_1) >= 30% of total   (≥30 ข้อ)
  SUM(tier_4) <= 25% of total   (≤25 ข้อ)
  
  Anchor Rule:
    1 document per set gets +5 above tier.max
    max 1 anchor per set
```

## Coverage Rules

> **v3.0** — ตัวเลขปรับตามฐาน 100 ข้อ/ชุด

| Rule | คำอธิบาย | Enforcement |
|---|---|---|
| **CR-1: Mandatory Topics** | Topic สำคัญต้องปรากฏในทุก Set | Hard — Fail ถ้าขาด |
| **CR-2: Minimum Unique Topics** | แต่ละ Set ต้องมี Topic ไม่ซ้ำ ≥ 70% (≥70 unique จาก 100 ข้อ) | Hard — Fail ถ้าต่ำกว่า |
| **CR-3: Cross-Set Rotation** | Topic เดียวกัน + Difficulty + Type เดียวกัน ห้ามปรากฏเกิน 5 Set | Soft — Warning (อัตโนมัติผ่านเสมอ เพราะมีทั้งหมด 5 Set) |
| **CR-4: Document Coverage** | ทุก Document ต้องปรากฏในทุก Set (minimum 5 ข้อ) | Hard — Fail ถ้าขาด |
| **CR-5: Section Sweep** | สำหรับ Tier 1: ทุก Section หลักต้องถูกวัดอย่างน้อย 1 ครั้งใน 5 Set รวมกัน | Soft — Warning |

## Duplicate Prevention Rules

> **v3.0** — ป้องกันข้อสอบซ้ำภายในและข้ามชุด (โครงสร้างเดิม ไม่เปลี่ยนตามจำนวนข้อ)

### ระดับการป้องกัน

| Level | ขอบเขต | กฎ | Enforcement |
|---|---|---|---|
| **L1** | ภายใน Set | Topic + Difficulty + Type เดียวกัน = ห้ามซ้ำ | Hard |
| **L2** | ภายใน Set | Section/มาตรา เดียวกัน ≤ 3 ข้อ/Set | Hard |
| **L3** | ข้าม Set | Topic + Difficulty + Type เดียวกัน ห้ามเกิน 5 Set | Soft (ผ่านอัตโนมัติ เพราะมี 5 Set ทั้งหมด) |
| **L4** | ข้าม Set | Concept/ประเด็นหลักต้องไม่ซ้ำเกิน 50% | Soft |
| **L5** | ข้าม Set | Scenario ต้องมีบริบทต่างกัน (ต่างสถานการณ์ ต่างตัวละคร) | Soft |

### Similarity Metric

```
calculate_similarity(question_a, question_b):

  topic_sim    = 1.0 if same_topic    else 0.0   (weight: 0.40)
  section_sim  = 1.0 if same_section  else 0.0   (weight: 0.20)
  type_sim     = 1.0 if same_type     else 0.0   (weight: 0.15)
  diff_sim     = 1.0 if same_diff     else 0.0   (weight: 0.10)
  keyword_sim  = jaccard(keywords_a, keywords_b)  (weight: 0.15)

  total = weighted_sum(above)

  >= 0.85 → BLOCK
  >= 0.70 → WARN
  <  0.70 → PASS
```

### ความใกล้เคียงข้อสอบราชการจริง

**จุดแข็ง:**
- น้ำหนักกฎหมายแม่บทสูงกว่ากฎหมายเสริม ตรงตามแนวทางคณะกรรมการจริง
- มี Scenario Question ซึ่งข้อสอบราชการระดับนักวิชาการนิยมออก
- ครอบคลุม Cross-Reference ระหว่างกฎหมาย กระจายอยู่ในทุกชุด
- มี Topic เกี่ยวกับ กทม. โดยเฉพาะ ซึ่งเป็นลักษณะเฉพาะของข้อสอบ กทม.

**ข้อสังเกตเพื่อปรับปรุง:**
- ข้อสอบราชการจริงอาจมี "ข้อใดกล่าวถูกต้อง" แบบ 4 ตัวเลือก ซึ่ง Blueprint Type "Memory" รองรับได้แต่อาจต้องเพิ่มสัดส่วน
- ข้อสอบจริงมักมีข้อ "ข้อใดไม่ใช่" ซึ่งต้องกำหนดใน Blueprint Type ย่อยเพิ่มเติม
- เนื่องจากลดจำนวนชุดจาก 10 เหลือ 5 แต่เพิ่มข้อ/ชุดเป็น 2.5 เท่า Topic เดิมบางหัวข้อจึงถูกนำมาใช้ซ้ำข้ามชุดมากกว่าเดิม (ดูตาราง Top 15) ควรพิจารณาเพิ่มคลังหัวข้อย่อยหากต้องการลดการซ้ำต่อไปอีก
